# NEONITE-555

## 简介

https://www.bilibili.com/video/BV11CGd6eEZ3/

NEONITE-555 是一款基于 ESP32-S3 的 AI 玩具机器人，具有放大化的 NE555P 芯片外观。项目集成了小智（Xiaozhi）智能对话功能，并通过八条联动腿实现舞蹈动作。

## 功能特性

1. **智能对话**：完整支持小智项目所有基础功能（大模型联网对话、自定义人物音色与记忆、查询天气、讲笑话等）
2. **舞蹈控制**：四舵机控制的八条联动腿，对应 NE555P 的 DIP-8 封装造型，每条腿约 ±45° 运动自由度，可实现走动、摇摆、跳舞等动作
3. **多种控制方式**：
   - 语音控制
   - 微信小程序 WebSocket 控制
   - 微信小程序 BLE 蓝牙控制

## 硬件规格

- **主控**：ESP32-S3-WROOM N16R8（16MB Flash，8MB PSRAM）
- **麦克风**：ZTS6672（原理图标注为 43434）
- **功放**：MAX98357AETE
- **充电管理**：IP5306
- **屏幕**：0.91 寸 OLED I2C 128×32
- **触控模块**：TTP223
- **舵机**：SG90-180度 × 4
- **电池**：默认 804050P 2000mAh（可自选）
- **喇叭**：默认 2030 8Ω 2W（可自选）

## 配置与编译

**前置要求**：ESP-IDF **v5.5.4**

### 快速编译步骤

```bash
# 1. 克隆仓库并进入目录
git clone https://github.com/78/xiaozhi-esp32.git
cd xiaozhi-esp32

# 2. 复制板级默认配置到项目根目录（只需一次）
cp main/boards/neonite-555/sdkconfig.defaults ./sdkconfig.defaults

#以上两个步骤也可以手动下载替换

# 3. 直接编译（target 和板子已在 sdkconfig.defaults 中预设）
idf.py build

# 4. 烧录与监控串口
idf.py flash monitor
```

> **注意**：
> - 固件较大，烧录时间约 19+13 秒，请耐心等待。
> - `sdkconfig.defaults` 已预设 `CONFIG_IDF_TARGET=esp32s3` 和 `CONFIG_BOARD_TYPE_NEONITE_555=y`，无需手动 `set-target` 或 `menuconfig`。

如需调整其他配置（如语言、唤醒词），可执行 `idf.py menuconfig` 后重新编译。

## 舞蹈文件说明

本项目支持自定义舞蹈编排，舞蹈文件通过 mmap 只读方式访问，已打包在固件中：

- `jiaqiwu.json` + `jiaqiwu.ogg`：嘉祺舞
- `chicken_nugget.json` + `chicken_nugget.ogg`：旋转鸡块

> **注意**：自定义编舞 JSON 和 OGG 音频需要重新编译固件才能更新。

## 注意事项

### 电气安全

1. **硅麦保护**：硅麦对液体敏感，焊接时**不建议**使用助焊剂、洗板水或清洁剂，可能造成损坏（收声孔在底部）
2. **上电前检查**：建议上电前测量 5V、3.3V 轨以及 BAT 是否存在短路
3. **焊接前测试**：建议在焊接排针前运行回环测试，检查硅麦与功放是否运行良好
4. **舵机保护**：过快的舵机动作会触发 IP5306 的大电流保护关机

### 软件使用

1. 固件较大，刷写时间约 19+13 秒
2. 自定义编舞 JSON、自定义 OGG 音频需要重新刷写固件
3. 首次使用蓝牙功能时，需要在手机上授权蓝牙权限

### 结构装配

1. **舵机校准**：安装舵机连接器前请先对舵机上电校准
2. **腿部干涉**：前后腿部在进行大角度动作时可能出现干涉，需掰动内部铁丝使其内外错开

## 常见问题

### 1. 蓝牙连接失败？

- 确认手机蓝牙已开启
- 如果是 Android 手机，需要授予小程序位置权限（蓝牙扫描需要）

### 2. 舞蹈动作不正常？

- 检查舵机是否已校准
- 检查腿部铁丝是否干涉
- 查看串口日志确认舞蹈文件是否成功加载

### 3. 如何添加新舞蹈？

- 准备舞蹈 JSON 文件和对应的 OGG 音频文件（格式参考 `assets/` 目录下的示例文件）
- 将文件放入 `main/boards/neonite-555/assets/` 目录
- 更新 `choreo_index.json` 添加舞蹈索引
- 重新编译并烧录固件

### 4. 设备黑屏死机？

可能原因：

- **用户自定义舞蹈舵机速度设置过快**：过快的舵机动作会触发 IP5306 大电流保护关机，建议降低舵机运动速度
- **电压过低**：电池电量不足时舵机启动瞬间压降过大，导致系统复位，请充电后重试
- **舵机堵转**：腿部结构干涉或机械卡死导致舵机堵转电流飙升，检查腿部装配是否正常
